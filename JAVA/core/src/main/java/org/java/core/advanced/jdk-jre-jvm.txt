jdk:
	Java Development Kit是Java Environment的核心组件，
提供编译，调试和执行Java程序所需的所有工具，可执行文件和二进制文件,
JDK是一个特定于平台的软件，这也是我们为Windows，Mac和Unix系统提供单独安装程序的原因.
 我们可以说JDK是JRE的超集，因为它包含带有Java编译器，Java调试器和Java核心类的JRE。
 
jvm:
 	JVM是java编程语言的核心。 当我们运行程序时，JVM负责将字节代码转换为特定于机器的代码,
JVM也依赖于平台，并提供核心Java功能，如内存管理，垃圾收集，安全性等.
JVM是可自定义的,我们可以使用java选项对其进行自定义，例如为JVM分配最小和最大内存.
JVM被称为虚拟，因为它提供的接口不依赖于底层操作系统和机器硬件。
 这种与硬件和操作系统的独立性使得java程序只写一次，然后直接在其他平台运行。

jre:
	JRE是JVM的实现，它提供了执行java程序的平台。 JRE由JVM和java二进制文件以及其
	他成功的执行任何程序的类组成。 JRE不包含任何开发工具，如java编译器，调试器等。
	如果要执行任何java程序，应该安装JRE，但我们不需要JDK来运行任何java程序。


总结:
1. JDK用于开发目的，而JRE用于运行java程序。
2. JDK和JRE都包含JVM，因此我们可以运行我们的java程序。
3. JVM是java编程语言的核心，提供平台独立性。	


Just-in-time Compiler (JIT)即时编译器:
有时我们听到这个术语，并且它是JVM的一部分，它让我们感到困惑。 
JIT是JVM的一部分，它通过同时编译相似的字节代码来优化字节码到机器特定的语言的
编译，从而减少了将字节码编译为机器特定语言所需的总时间。
